ES6 模块化

ES6 引入了原生的模块系统，使用 import 和 export 关键字。这是 JavaScript 语言层面的模块化方案，相比 CommonJS 和 AMD，具有更好的静态分析和优化能力。

导出方式

// 命名导出（Named Export）


export const name = 'value';


export function func() {}


export class MyClass {}
// 或者统一导出


export { name, func, MyClass };
// 默认导出（Default Export）


export default function() {}


export default class {}
// 混合导出


export const named = 'value';


export default function() {}

导入方式

// 命名导入


import { name, func } from './module';


import { name as myName } from './module'; // 重命名
// 默认导入


import defaultExport from './module';
// 混合导入


import defaultExport, { named } from './module';
// 全部导入


import * as module from './module';
// 动态导入（返回 Promise）


import('./module').then(module => {


  // 使用模块


});

与 CommonJS 的区别

  • 加载时机: ES6 模块在编译时确定依赖，CommonJS 在运行时加载
  • 值引用: ES6 模块输出的是值的引用，CommonJS 输出的是值的拷贝
  • 静态分析: ES6 模块支持静态分析，可以进行树摇优化
  • 严格模式: ES6 模块自动采用严格模式

优势

  • 静态分析: 编译时就能确定模块依赖关系
  • 树摇优化: 打包工具可以移除未使用的代码
  • 循环依赖检测: 可以在编译时发现循环依赖问题
  • 更好的 IDE 支持: 提供更好的代码提示和跳转

使用场景

  • 现代前端项目（React、Vue 等）
  • 需要代码分割和懒加载
  • 需要 Tree Shaking 优化
  • 浏览器原生支持（需要 type="module"）
