前端性能优化

前端性能优化是提升用户体验的关键，涉及资源加载、渲染性能、网络请求等多个方面。

资源优化

  • 代码压缩和混淆: 减小文件体积，使用 UglifyJS、Terser 等工具
  • Tree Shaking: 移除未使用的代码
  • 图片优化:
  • 使用 WebP、AVIF 等现代格式
  • 图片懒加载（Intersection Observer）
  • 响应式图片（srcset）
  • 图片压缩和 CDN
  • CDN 加速: 使用内容分发网络加速资源加载
  • 资源预加载:
  • 预加载关键资源
  • 预取未来可能需要的资源
  • DNS 预解析

渲染优化

  • 减少重排重绘:
  • 使用 transform 和 opacity 做动画（触发合成层）
  • 批量 DOM 操作
  • 使用 DocumentFragment
  • 虚拟滚动: 只渲染可见区域的列表项
  • 代码分割:
  • 路由级别的代码分割
  • 组件懒加载（React.lazy、动态 import）
  • 防抖节流: 限制函数执行频率
  • 使用 Web Worker: 将耗时任务移到 Worker 线程

网络优化

  • HTTP/2: 多路复用、头部压缩
  • 资源压缩: Gzip、Brotli 压缩
  • 缓存策略:
  • 强缓存（Cache-Control、Expires）
  • 协商缓存（ETag、Last-Modified）
  • Service Worker 缓存
  • 减少 HTTP 请求:
  • 合并文件
  • 使用雪碧图（CSS Sprites）
  • 内联小资源

首屏优化

  • 关键 CSS 内联
  • 骨架屏
  • 服务端渲染（SSR）
  • 预渲染（Prerendering）

性能监控

  • 使用 Performance API
  • Lighthouse 性能评分
  • Web Vitals 指标（LCP、FID、CLS）
