前端安全防护

前端安全是 Web 开发中不可忽视的重要环节。了解常见的安全威胁和防护措施，可以有效保护应用和用户数据。

常见攻击类型

1. XSS（跨站脚本攻击）

攻击者注入恶意脚本到网页中，当其他用户浏览时执行。

类型:

  • 存储型 XSS: 恶意脚本存储在服务器（如评论、留言）
  • 反射型 XSS: 恶意脚本通过 URL 参数反射到页面
  • DOM 型 XSS: 通过修改 DOM 环境执行恶意代码

防护措施:

  • 输入验证和过滤: 对用户输入进行严格验证
  • 输出编码: 使用 textContent 而非 innerHTML
  • CSP（内容安全策略）: 限制脚本执行来源
  • HttpOnly Cookie: 防止 JavaScript 访问敏感 Cookie
  • 使用框架的转义机制: React、Vue 等框架自动转义

示例:

// 危险


el.innerHTML = userInput; // 可能执行恶意脚本
// 安全


el.textContent = userInput; // 自动转义

2. CSRF（跨站请求伪造）

攻击者诱导用户执行非本意的操作。

防护措施:

  • CSRF Token: 服务器生成 token，客户端携带验证
  • 验证 Referer: 检查请求来源
  • SameSite Cookie: 设置 Cookie 的 SameSite 属性
  • 双重 Cookie 验证: 客户端和服务端都验证 Cookie

3. 点击劫持（Clickjacking）

通过 iframe 覆盖透明层，诱导用户点击。

防护措施:

  • X-Frame-Options: 设置 DENY 或 SAMEORIGIN
  • CSP frame-ancestors: 更灵活的 iframe 控制

4. SQL 注入（虽然主要是后端问题）

前端应该避免拼接 SQL 语句，使用参数化查询。

5. 敏感信息泄露

  • 不要在客户端存储敏感信息（密码、密钥等）
  • 不要在代码中硬编码 API 密钥
  • 使用环境变量管理配置

防护措施总结

输入处理

  • 严格验证用户输入（白名单验证）
  • 过滤和转义特殊字符
  • 限制输入长度和格式

输出处理

  • 对所有输出进行编码
  • 使用安全的 DOM API
  • 避免使用 eval()、innerHTML 等危险方法

传输安全

  • HTTPS: 使用加密传输
  • HSTS: 强制使用 HTTPS
  • 安全 Cookie: 设置 Secure、HttpOnly、SameSite 属性

内容安全策略（CSP）

<meta http-equiv="Content-Security-Policy" 


      content="default-src 'self'; script-src 'self' 'unsafe-inline';">

其他安全措施

  • 依赖安全: 定期更新依赖，检查已知漏洞
  • 代码混淆: 保护业务逻辑（但不能完全防止）
  • 权限控制: 前端权限验证（后端必须再次验证）
  • 日志监控: 记录异常行为，及时发现攻击

安全开发建议

  • 永远不要信任客户端输入
  • 使用成熟的安全库和框架
  • 定期进行安全审计
  • 关注 OWASP Top 10 安全风险
  • 实施最小权限原则
